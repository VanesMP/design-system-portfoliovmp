# Design System

A personal design system built to power two distinct creative portfolios: **Dev** and **Art**.
Each portfolio has its own visual identity, color palette.
All driven by a shared token architecture and a single component library.

---

## What is this?

This design system is the single source of truth for the UI of my portfolio projects.
Rather than duplicating styles across codebases, every visual decision : colors, typography,
spacing and shadows, lives here as a **design token**, generated by Style Dictionary and consumed
as CSS custom properties.

Swap a theme class on `:root`, and the entire interface adapts. No component logic changes.
No prop drilling. Just CSS doing its job.

## Token architecture

Design tokens are defined as JSON files, one per theme, and compiled by **Style Dictionary**
into CSS custom properties scoped to a root class:

```css
:root.dev-dark  { --color-background: #00241D; ... }
:root.dev-light { --color-background: #F6F8F2; ... }
:root.art-dark  { --color-background: #1b0000; ... }
:root.art-light { --color-background: #F9F1EC; ... }
```

Token categories: `color` · `shadow` · `typography` · `spacing` · `border-radius`

Switching theme is a single class change on `:root`. All components adapt automatically.
